    Buttons and links

    GitHub

    Call-to-action action button

    Use a main call-to-action button when the user is moving forward in a path, conveying a sense of purpose and urgency.
    For example Register now, Contact us or Join the mailing list.

    • avoid excessively long button text
    • make sure the button text describes an action
    • try to have as few call-to-action buttons per page as possible - just one is best.
    Main call-to-action buttons on NICE.org look like this:

    HTML code for a call to action button

    <a href="/register/register-now" class="btn btn-cta">Register now</a>

    Standard button

    Use the standard button for normal actions, do something within the page, typically to reveal more information.
    For example View guidance consultations, List of guidelines or Advice listing.

    • avoid excessively long button text
    • make sure the button text describes an action.
    Standard buttons on NICE.org look like this:

    HTML code for a standard button

    <a href="⁄guidance⁄inconsultation"" class="btn">View guidance consultations/a>

    Key link

    Use the key link for more information, read more or find out more, typically to read more information.
    For example Read more about NICE guidelines, View the case study or Learn more.

    • avoid excessively long link text
    • make sure the link text describes an action.
    Standard buttons on NICE.org look like this: Read more about NICE guidelines

    HTML code for a key link link

    <a href="⁄About⁄What-we-do/⁄Our-Programmes⁄NICE-guidancee⁄NICE-guidelines" class="key-link">Read more about NICE guidelines<⁄a>

    Text link

    Use text links for any other links within the text.
    Avoid opening content in a new window if you can, as it can be very confusing for some people (particularly screen reader users).

    • generalised link within the text
    • not an action
    • navigational
    • make them describe the thing they link to.
    Text links on NICE.org look like this: NICE field team

    HTML code for a text link

    <a href="www.nice.org.uk">Example text link<⁄a>

    Anchor links

    Use anchor links to jump to a specific location on a page.
    For example Down to Directors, Down to Executive directors or Back to top.

    The anchor tag will consist of two HTML elements. First, you'll want to create the link.

    <a href="#directors">Directors<⁄a>

    The second part of an anchor tag is the actual anchor. The anchor should be placed at the beginning of the line where you want to start reading after you jump similar to:

    <h2><a id="director"></a>Directors</h2>

    Download links

    Use PDF download links when displaying large numbers of PDF documents for download, for example:
    Policies and procedures or the Citizens Council pages

    • display a PDF icon, tell the user up front that the link they are about to click is a PDF
    • show a file size: it is best practice to show your users what they can expect and how long it may take them.
    Download links on NICE.org look like this:
    • NICE charter PDF 156 Kb

    HTML code for a download link

    <div class="panel-body">
<ul class="media-list">
<li class="media media-resource">
<div class="media-body">
<h6 class="media-heading resource-link"><a href="#">
<i class="icon-nice-download"></i> NICE charter</a>
<span class="stats"> <span class="data-size">PDF 156 Kb</span> </span></h6>
</div>
</li>
</ul>
</div>
</div>

    Lists

    Long-entry bulleted lists

    Use a list to display a set of terms, phrases, or statements clearly.

    Long-entry bullet lists on NICE.org look like this:
    • Each entry usually consists of at least 1 complete sentence and ends with a full stop. But, you can also use the long-entry list style if you have a list in which the items are long but aren't complete sentences.
    • The lead-in line can be a sentence or part of a sentence, and it should always end with a colon. Bullets do not have to run on from the lead-in line
    • Long-entry lists do not necessarily need a lead-in line; the bullets can stand alone.

    Short-entry bulleted list

    A short-entry list is introduced with a colon, and the bulleted items.

    Short-entry bullet lists on NICE.org look like this:
    • start with a lower-case letter
    • contain items with only a few words
    • usually run on from the lead-in line (unless phrases such as 'all of' or 'any of' are added to the lead-in line)
    • never have a new sentence starting within them
    • have a full stop at the end of the last item only.

    HTML code for a bullet list

    <ul>
<li>...</li>
<li>...</li>
<li>...</li>
</ul>

    Numbered list

    Use numbered lists when you want to indicate a hierarchy, or the numbers or letters form part of a specific grading system.

    Numbered lists on NICE.org look like this:
    1. the safety of the procedure
    2. whether it works well enough for routine use
    3. whether special arrangements are needed for patient consent.

    HTML code for a numbered list

    <ol>
<li>...</li>
<li>...</li>
<li>...</li>
</ol>

    Highlighted numbered list

    Use the highlighted numbered list when you want to emphasize a step by step process and make it a feature of the content.

    Highlighted numbered lists on NICE.org look like this:
    1. Guideline topic is referred
    2. Stakeholders register interest
    3. Final guideline produced
    4. Guidance issued

    HTML code for a highlighted numbered list

    <ol class="alt">
<li>...</li>
<li>...</li>
<li>...</li>
</ol>

    Panels

    Use a panel in the right hand column of the page layout, usually a panel is used to show a call-to-action button such as Register here or Contact us

    Panels on NICE.org look like this:

    Contracts Finder registration

    Details of tenders available and other organisations interested in working with NICE or providing goods or services will be listed on the Contracts Finder website.

    Register here

    HTML code for a panel

    <div class="panel panel-default">
<div class="panel-heading">
<h4>Contracts Finder registration</h4>
</div>
<div class="panel-body">
<p>Details of tenders available and other organisations interested in working with NICE or providing goods or services will be listed on the Contracts Finder website.</p>
<p><a class="btn btn-cta" href="https://online.contractsfinder.businesslink.gov.uk">Register here</a></p>
</div>
</div>
