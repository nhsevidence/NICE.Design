Grids, type and vertical rhythm

Overview

Firstly, this guide is a work in progress and is hopefully a starting point for us to collaborate and refine how we prototype/design with grids and type.

In the absence of visual elements — arguably the case for many of NICE products — establishing foundations for typography underpins the content we output. This guide is designed to address vertical rhythm and hierarchy, and can be used to promote consistency of type.

Grids

Baseline

On the basis of using a 24px baseline grid (a common browser default), we can establish a predictable and robust vertical rhythm which should dictate line heights, define hierarchy and add consistent spacing between elements.

Columns

The ‘magic number’ 24 also extends to our column grid, i.e. column widths of 72px (3 x 24), and margins of 24px. This gives us a maximum site width of 1176px. Visit Grid Calculator to see this in action. You can also download this grid as a PNG, Photoshop action, or Illustrator template for conceptual design work.

Based on typical breakpoints, we could set our maximum site widths as follows:

  • Large devices (laptops, desktops):1176px
  • Medium devices (large tablets, netbooks):960px
  • Small devices (small tablets):748px
  • Extra small devices (smartphones):100% fluid
  • Tiny devices (smart watches):Are you having a laugh?

Typography

Fonts

For the sake of legibility, we use Lato regular, bold and their respective italics as the standard weights. Lighter weights are also available, but are less readable on screen, particularly for Windows users.

Typographic scale

Scale should be used to establish a clear and contrasting information hierarchy. To balance density of content and reading comfort, these are the optimal type sizes and styles that — along with the baseline grid — work well together for screen use.

Heading styles

Heading 1 — Diagnosis of drug allergy in adults

Style

  • Font size: 48px (3rem)
  • Line height: 1
  • Weight: 400/regular

Context

Use once per page, typically reserved for the page/article title.

Heading 2 — AaBbCcDdEeFfGgHh

Style

  • Font size: 36px (2.25rem)
  • Line height: 1.333333333333333
  • Weight: 400/regular

Context

Used for any form of important page-level header, such as a section title. More than one may be used per page. Consider using an h2 unless you need a header level of less importance, or as a sub-header to an existing h2 element.

Heading 3 — AaBbCcDdEeFfGgHh

Style

  • Font size: 24px (1.5rem)
  • Line height: 1
  • Weight: 400/regular

Context

Used for any form of page-level header which falls below the h2 header in a document hierarchy, such as a section sub-title.

Heading 4, 5 and 6 — AaBbCcDdEeFfGgHh

Style

  • Font size: 16px (1rem)
  • Line height: 1.5
  • Weight: 400/regular

Context

Use for anything that requires a subtle label or definition without dominating the main copy. Discourage excessive levels of [visual] hierarchy by having identical styles for H4/H5/H6.
